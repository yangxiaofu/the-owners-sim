## 13. Troubleshooting Display Issues

### 13.1 Common Display Problems

```
Problem: Lines don't align
  Solution: Check terminal is â‰¥80 columns wide
  Test: Verify width with: shutil.get_terminal_size().columns

Problem: Colored text not showing
  Solution: Check ANSI color support
  Test: Verify with: sys.stdout.isatty()

Problem: Unicode characters showing as boxes
  Solution: Terminal doesn't support Unicode
  Solution: Set unicode_fallback: true in config

Problem: Text wrapping mid-line
  Solution: Terminal width too narrow
  Solution: Resize to â‰¥80 columns or check terminal emulator settings

Problem: Overlapping text
  Solution: ANSI codes not stripped properly when calculating width
  Solution: Use strip_ansi_codes() before len() calculations

Problem: Pause prompt not appearing
  Solution: Buffered stdout not flushing
  Solution: Use sys.stdout.flush() after print()
```

## 15. Rendering Checklist

```
Before rendering a screen:
- [ ] Check terminal width (detect and adapt)
- [ ] Check color support (enable/disable ANSI)
- [ ] Check Unicode support (fallback to ASCII)
- [ ] Strip ANSI codes before calculating text width
- [ ] Verify total row width matches specification
- [ ] Flush stdout after print()
- [ ] Test with actual terminal (not IDE console)
- [ ] Verify scroll distance (should fit on screen)
- [ ] Test with narrow terminal (70-80 columns)
- [ ] Test on different platforms (macOS, Linux, Windows)
```
